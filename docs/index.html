Appointment Booking API

A highly scalable, transaction-safe RESTful API for managing service providers, automated availability generation, and customer bookings.

Endpoints

  • POST /api/register/
  • POST /api/providers/
  • POST /api/generate-slots/
  • GET /api/slots/
  • POST /api/book/
  • GET /api/appointments/

Architecture & Base URL

Built with Django REST Framework and AWS RDS PostgreSQL. This microservice architecture uses row-level locking to ensure zero double-bookings during high concurrency.

Base URL: http://api-env.eba-45cakfm9.us-east-1.elasticbeanstalk.com

Authentication

All endpoints (except registration) require a valid API key passed in the headers. Organizations are isolated; keys only grant access to data owned by that organization.

X-API-KEY: <your_api_key>
Content-Type: application/json

API Reference

POST

/api/register/

Onboards a new organization. Returns the api_key required for all other requests.

{
  "organization_name": "Dublin Health",
  "username": "admin_user",
  "password": "secure_password"
}
POST

/api/generate-slots/

Automated Schedule Generator

Bulk-creates 30-minute availability slots for a provider. It skips existing slots to prevent duplicates.

Payload

{
  "provider_id": 1,
  "date": "2026-03-15",
  "start_time": "09:00",
  "end_time": "17:00"
}
POST

/api/providers/

Registers a new service provider (doctor/staff) within your organization.

{ "name": "Dr. smith" }
GET

/api/slots/

Fetch available (unbooked) time slots. Use these slot_id values for the booking endpoint.

GET /api/slots/?provider_id=1&date=2026-03-15
POST

/api/book/

Finalizes an appointment. This endpoint is idempotent and handles concurrency via database-level locking.

{
  "slot_id": 105,
  "customer_name": "John Doe",
  "customer_email": "john@example.com"
}
